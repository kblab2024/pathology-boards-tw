## 2. 你最常會做的事

### 修改一份 handbook，然後 render 確認
```powershell
# 編輯 handbook/03-mock-exam-key.md
python tools/render/render.py --file 03
# 打開 rendered/03-mock-exam-key.pdf 視覺檢查
```

### 改 CSS 樣式
```powershell
# 編輯 assets/styles/03_components.css
python tools/render/render.py --master
# 或 --file 09 試一份卡片風格
```

### 加一份新章節
1. `handbook/10-foo.md` — 加 H1 + AI 揭露 blockquote + 內容
2. `tools/render/config.py` 的 `CHAPTERS` list 追加 dict
3. `CHAPTER_ACCENT` 加新色
4. `python tools/render/render.py --all`

## 4. Render pipeline

```
handbook/*.md ──┐
                ├─ markdown_to_html.py (引用剝除 + audit)
handbook/*.csv ─┘                │
                                 │
                  Jinja templates (cover/colophon/divider/body/toc/back)
                                 │
                  Playwright (headless Chromium, per-section)
                                 │
                  pypdf 合併 → rendered/<num>-<slug>.pdf
                                 │
                  Master Edition: 合併全部章節 + cover/colophon/toc/back
```

**核心安全網**：
- `markdown_to_html.audit_path_references(html)` 掃 `<body>` 內容，命中 `.md/.json/.csv` 路徑或 `handbook/`、`tools/render/` 之類就 raise。在 render.py 的 `_assert_clean` 強制檢查。**不要拆掉這個 audit。**

## 5. 不要犯的錯

1. **不要在重新 render 後把鎖死的 AI 揭露 blockquote 拿掉。** 那是使用者明確要求保留的。
2. **不要為了「乾淨」把封面/頁腳的 Claude Code 署名刪掉。** 同上。
3. **不要在 PDF 內文加「想了解更多請見 `04-confusion-pairs.md`」這類 cross-reference。** 改用「請見章節 04」之類的非路徑指稱。
4. **不要把 `data/*.json` 改掉。** 那是不可變的官方題目原始檔。
5. **不要安裝新的重型依賴**（pandoc、TeX、wkhtmltopdf、playwright 以外的瀏覽器引擎）。
6. **不要用 `--amend`/`--force` 覆蓋已 commit 的 AI 揭露 history。**
7. **不要把 `rendered/*.pdf` 從 git 排除**（除非使用者明說）— 這些 PDF 是發佈成品，應該跟著 commit。
8. **CJK 文字抽取亂碼是 pypdf 的已知問題**，PDF 本身顯示是對的。不要因此改字型。
